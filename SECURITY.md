# Security Policy

This is the default security policy for `azmara-labs` repositories. It applies to any repository in this org that doesn't define its own `SECURITY.md`.

## Reporting a Vulnerability

Please report security vulnerabilities privately - **do not** open a public issue.

**Email:** gazmutech@gmail.com

Include:
- A description of the vulnerability and its potential impact
- Steps to reproduce (proof-of-concept code, if applicable)
- The affected repository, file(s), and version/commit

We aim to acknowledge reports within 3 business days and to keep you updated as the issue is triaged and fixed.

## Supported Versions

Only the latest published release of each package/app is supported with security fixes. Please upgrade rather than requesting a backported patch for an older version.

## Scope

This policy covers code maintained in `azmara-labs` repositories. Vulnerabilities in third-party dependencies should ideally be reported upstream to their own maintainers, but we're happy to be notified as well so we can track and patch on our end.
